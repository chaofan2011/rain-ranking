# 全国降雨量排行推送系统

基于 Node.js 的全国降雨量/体感温度排行系统，自动采集天气数据并推送到飞书群。

## 功能特性

- 每日自动采集全国 33 个城市降雨数据
- 每小时推送全国体感温度排行（08:00-20:00）
- 飞书卡片消息推送（降雨排行、体感温度排行、采集告警）
- 支持手动触发数据采集
- 健康检查接口
- 定时任务日志记录

## 技术栈

- Node.js + Express
- MySQL
- node-cron（定时任务）
- dayjs（时间处理）
- axios（HTTP 请求）
- 和风天气 API
- 飞书 Webhook

## 安装部署

### 1. 克隆代码

```bash
git clone https://github.com/chaofan2011/rain-ranking.git
cd rain-ranking
```

### 2. 安装依赖

```bash
npm install
```

### 3. 配置环境变量

复制 `.env.example` 为 `.env`，填写配置：

```bash
# 数据库配置
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=rain_db

# 和风天气配置
QWEATHER_KEY=your_key
QWEATHER_HOST=your_host

# 飞书配置
FEISHU_WEBHOOK=your_webhook_url
```

### 4. 初始化数据库

```bash
mysql -u root -p rain_db < rain_db.sql
```

### 5. 启动服务

```bash
# 开发模式
node app.js

# 生产模式（PM2）
pm2 start app.js --name rain-ranking
```

## API 接口

### 健康检查

```
GET /health
```

返回：

```json
{
  "status": "ok",
  "uptime": "2天3小时",
  "lastCollect": "成功",
  "time": "2026-06-08 14:30:00"
}
```

### 查询降雨排行

```
GET /rain/yesterday?date=2026-06-07
```

### 手动触发采集

```
GET /collect/yesterday?date=2026-06-07
```

## 定时任务

| 时间 | 任务 | 说明 |
|------|------|------|
| 每天 05:00 | 降雨数据采集 | 采集昨日降雨数据并推送排行榜 |
| 每天 08:00-20:00 | 体感温度排行 | 每小时推送全国体感温度 Top10 |

## 飞书推送

### 推送类型

| 类型 | 颜色 | 触发条件 |
|------|------|----------|
| 降雨排行 | 蓝色 | 每天 05:00 采集完成后 |
| 采集告警 | 红色 | 有城市采集失败时 |
| 体感温度 | 橙色 | 每小时（08:00-20:00） |

## 一键部署

配置 SSH 密钥后，执行：

```bash

```

脚本会自动：
1. 同步代码到服务器
2. 安装依赖
3. 重启服务
